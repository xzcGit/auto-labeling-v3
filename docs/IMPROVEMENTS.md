## P1 快捷键与操作效率

> **目标**: 提升标注效率，减少鼠标操作依赖
> **现状**: 仅 4 个菜单快捷键 (Ctrl+N/O/E/Q)，无标注操作快捷键，无批量操作

### P1-1 完善快捷键系统

- **状态**: ✅ 大部分已完成
- **涉及文件**: `src/ui/label_panel.py`, `src/ui/canvas.py`
- **已实现** (label_panel.py:470-520):
  - 工具切换: `W` 画框, `K` 关键点, `V` 选择
  - 图片导航: `A/←` 上一张, `D/→` 下一张
  - 标注操作: `Delete` 删除, `Ctrl+Z` 撤销, `Ctrl+Y` 重做
  - 标注确认: `Space` 确认选中, `Ctrl+Space` 确认全部
  - 视图控制: `Ctrl++/-/0` 缩放
  - 其他: `Ctrl+S` 保存, `F5` 刷新, `Shift+A` 自动标注, `Ctrl+Shift+A` 批量自动标注
- **待完善**: `Ctrl+C/V` 标注复制粘贴 (→ 合并至 P1-3)

### P1-2 批量操作支持

- **状态**: ✅ 已完成
- **涉及文件**: `src/ui/file_list.py`, `src/ui/label_panel.py`, `src/app.py`
- **改进内容**:
  - FileList 支持多选 (Ctrl+Click, Shift+Click)
  - 批量确认/取消确认标注
  - 批量删除标注
  - 批量修改类别 (选中标注 → 右键 → 更改类别)
- **验收标准**: 支持多选操作，批量操作有进度提示

### P1-3 标注复制与锁定

- **状态**: ✅ 已完成
- **涉及文件**: `src/ui/canvas.py`, `src/ui/label_panel.py`

- **验收标准**: 复制粘贴可用，锁定状态有视觉提示

### P1-4 标注统计面板

- **状态**: ✅ 已完成
- **涉及文件**: `src/ui/properties.py` (扩展), 新建 `src/ui/stats_panel.py`
- **现状**: 仅 properties.py 中显示 "标注: X | 确认: X | 待确认: X"

## P2 日志与错误处理

> **目标**: 提升可维护性，问题可追溯
> **现状**: 整个 src/ 仅 12 处 logger 调用 (7 error + 3 warning + 1 exception + 1 info)，9 处宽泛 except Exception

### P2-1 统一日志框架

- **状态**: ✅ 已完成
- **涉及文件**: 新建 `src/utils/logging.py`, 全局各模块
- **改进内容**:
  - 建立统一日志配置 (console + rotating file)
  - 日志文件存放于 `~/.autolabel/logs/`
  - 格式: `[时间] [级别] [模块] 消息`
  - 关键路径添加日志:
    - 项目创建/打开/保存
    - 模型加载/训练启动/完成
    - 导出操作
    - 标注保存
    - 自动标注 (每张图)
- **验收标准**: 所有关键操作有日志记录，日志文件自动轮转 (最大 10MB × 5 个)

### P2-2 细化异常处理

- **状态**: ✅ 已完成
- **涉及文件**: `src/app.py` (9处), `src/utils/workers.py` (2处)
- **改进内容**:
  - `except Exception` → 细化为具体异常类型:
    - 文件操作: `FileNotFoundError`, `PermissionError`, `OSError`
    - JSON 解析: `json.JSONDecodeError`, `KeyError`
    - 模型加载: `RuntimeError`, `FileNotFoundError`
    - 训练: `RuntimeError`, `ValueError`
  - 每个 except 块添加有意义的日志 (含上下文参数)
  - 用户级错误提示区分: 可恢复 vs 需要操作
- **验收标准**: 无宽泛 except Exception，异常信息包含上下文

### P2-3 线程安全修复

- **状态**: ✅ 已完成
- **涉及文件**: `src/utils/workers.py`
- **现状**:
  - `TrainWorker.cancel()`: 无锁访问 `self._trainer` (line 38-39)
  - `BatchPredictWorker`: `self._cancelled` 标志跨线程读写无同步 (line 94, 100)
- **改进内容**:
  - 为 `_cancelled` 使用 `threading.Event` 替代裸布尔值
  - `_trainer` 访问加 `QMutex` 保护
- **验收标准**: 并发场景无竞态，有对应单元测试

### P2-4 输入校验完善

## P3 Canvas 性能优化

> **目标**: 密集标注场景 (>100 bbox) 下保持流畅
> **现状**: paintEvent 全量遍历所有标注重绘, 764 行无分层渲染

### P3-1 脏区域重绘

- **状态**: ✅ 已完成 (合并至视口裁剪优化)
- **涉及文件**: `src/ui/canvas.py`
- **改进内容**:
  - 仅重绘变化区域 (选中/拖动的标注)
  - 使用 `update(QRect)` 代替 `update()` 全量刷新
  - 图片层缓存为 QPixmap，标注变化时不重绘图片
- **验收标准**: 拖动单个标注时不触发全量重绘

### P3-2 图片缓存增强

- **状态**: ✅ 已完成
- **涉及文件**: `src/utils/image.py`
- **现状**: LRU 缓存 16 张原图
- **改进内容**:
  - 缩略图缓存 (用于 FileList 预览, 如果后续添加)
  - 预加载策略优化: 根据浏览方向预加载
  - 内存监控: 缓存总大小上限 (如 512MB)，超限时自动缩减
- **验收标准**: 大项目 (1000+ 图) 浏览不卡顿

### P3-3 大批量标注渲染优化

- **状态**: ✅ 已完成
- **涉及文件**: `src/ui/canvas.py`
- **改进内容**:
  - 视口裁剪: 仅绘制可见区域内的标注
  - 缩放级别分层: 缩小到一定程度时简化标注绘制 (不画标签文字、简化关键点)
  - 选中标注单独图层，避免遮挡
- **验收标准**: 200+ 标注时帧率 ≥30fps
